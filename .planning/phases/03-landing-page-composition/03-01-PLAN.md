  <action>
    Create `assets/kanva-hero.js` as a vanilla JS IIFE (per D-01, no module system, matching Horizon's asset loading pattern).

    The JS must:
    1. Wrap in an IIFE: `(function () { ... })()`
    2. Use `document.querySelectorAll('.kanva-hero')` to support multiple hero instances
    3. For each hero instance:
       - Query `.kanva-hero__slides` container
       - Query all `.kanva-hero__dot` buttons
       - Track `current` slide index (starts at 0), `total` count from dots.length
       - `goTo(n)` function: sets `current = (n + total) % total`, applies `slides.style.transform = 'translateX(-' + (current * 100) + '%)'`, toggles `active` class on dots
       - `start()` function: sets `timer = setInterval(function () { goTo(current + 1) }, 5000)` (5 second auto-advance, within Claude's discretion)
       - `reset()` function: `clearInterval(timer); start()`
       - Attach click listeners to each dot: `goTo(+d.dataset.index); reset()`
       - If `total > 1`, call `start()` to begin auto-advance
    4. No semicolons at line ends (matching Horizon JS convention from CONVENTIONS.md)
    5. Use `var` declarations inside the IIFE (not const/let) for maximum browser compatibility in the non-transpiled context, OR use `const/let` if all other Horizon assets use modern syntax (check conventions)
    6. No external dependencies

    The IIFE pattern is idempotent — safe for Shopify's theme editor re-rendering (Pitfall 3 from research).
